# Social-Feed
Social media feed equivalent - education project ICH

# Цель: Создать аналог ленты социальной сети, где можно просматривать посты, комментарии и информацию об авторах.

Описание:
Приложение имитирует социальную сеть, загружая посты, комментарии к ним и данные пользователей. Пользователь может просматривать посты, открывать комментарии и "создавать" новые посты.

Технические требования:

Загрузка и объединение данных:

Сделать запросы к:

/posts (список постов)
/users (список пользователей)
/comments (список всех комментариев)

Объединить данные так, чтобы каждый пост в массиве содержал:
Свойства поста (id, title, body, userId).
Объект автора (user) с его именем и email.
Массив комментариев (comments), относящихся к этому посту.

Отобразить посты в виде карточек. В карточке поста должен быть:
Заголовок (title).
Текст поста (body).
Имя автора.
Количество комментариев (например, "Комментарии (5)").
Кнопка "Показать/Скрыть комментарии".

Работа с комментариями:
При нажатии на "Показать комментарии" под постом должен динамически появляться блок со списком комментариев к этому посту.
Для каждого комментария отображать: имя (из свойства name) и текст (из свойства body).
При повторном нажатии комментарии должны скрываться.

"Создание" нового поста:
Реализовать форму для создания нового поста с полями: "Заголовок" и "Текст".

При отправке формы:
(Опционально) Отправить POST-запрос на /posts.
Создать новый объект поста (сгенерировать временный ID, использовать данные из формы).
Добавить этот пост в начало локального массива данных и немедленно отобразить его в ленте.
Примечание: так как у нового поста нет userId, можно временно подставлять первого пользователя из списка.

Код должен быть структурирован: fetchInitialData(), createPostCard(postData), toggleComments(postId).

API Endpoints:

https://jsonplaceholder.typicode.com/posts
https://jsonplaceholder.typicode.com/users
https://jsonplaceholder.typicode.com/comments
